Buttons allow users to take actions, make choices, and trigger events with a single click or tap.

The Button component follows WAI-ARIA guidelines:

- Uses native `<button>` elements for proper keyboard interactions
- Includes appropriate ARIA attributes when needed
- Maintains sufficient color contrast in all variants
- Provides focus styles for keyboard navigation

## Best Practices

- Use clear, action-oriented text that describes what happens when the button is clicked
- Choose the appropriate variant based on the button's importance
- Primary buttons should be used for the main action in a form or page
- Limit the number of primary buttons on a page to avoid confusion
- Keep button text concise (2-3 words is ideal)
- Use icons to enhance meaning, not replace text (except for well-known icons)

## Implementation Notes

The Button component is built using Tailwind CSS for styling. The styles adjust automatically based on the selected theme. */}
