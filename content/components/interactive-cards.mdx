# Interactive Cards Component

Interactive cards are versatile components that display content in an organized, visually appealing way. They support expandable details, smooth animations, and can be customized for various content types including products, blog posts, events, and more.

## Key Features

- **Expandable Details**: Click any card to view full details in an overlay
- **Smooth Animations**: Built with Framer Motion for fluid transitions
- **Responsive Design**: Works seamlessly across all device sizes
- **Multiple Content Types**: Support for products, articles, events, and custom content
- **Rich Metadata**: Display stats, dates, locations, and other contextual information
- **Interactive Elements**: Hover effects, social links, and action buttons
- **Keyboard Navigation**: ESC key support for closing expanded views

## Customization Options

### Card Data Structure

```typescript
interface CardItem {
  id: number;
  title: string;
  subtitle: string;
  description: string;
  image: string;
  category?: string;
  tags?: string[];
  stats?: {
    views?: number;
    likes?: number;
    shares?: number;
    bookmarks?: number;
  };
  metadata?: {
    date?: string;
    location?: string;
    duration?: string;
    author?: string;
  };
  actions?: {
    primary: string;
    secondary?: string;
  };
  socials?: {
    linkedin?: string;
    github?: string;
    twitter?: string;
    mail?: string;
  };
}
```

### Available Props

- `cards`: Array of CardItem objects
- `type`: Optional card type for styling variations
- `onCardClick`: Custom handler for card interactions
- `expandable`: Whether cards can be expanded (default: true)
- `gridCols`: Custom grid layout (default: responsive)

### Styling Variants

The component automatically adapts its styling based on the content type and available data:

- **Product Cards**: Emphasize pricing, ratings, and purchase actions
- **Blog Cards**: Highlight reading time, author, and engagement metrics
- **Event Cards**: Focus on dates, locations, and registration
- **Profile Cards**: Feature social links and contact information

### Animation Configuration

Built with Framer Motion, the component includes:

- **Layout Animations**: Smooth transitions between grid and expanded views
- **Hover Effects**: Subtle scale and shadow changes
- **Loading States**: Stagger animations for card appearance
- **Gesture Support**: Swipe to close on mobile devices

## Accessibility Features

- **Keyboard Navigation**: Full keyboard support with ESC to close
- **Screen Reader Friendly**: Proper ARIA labels and semantic HTML
- **Focus Management**: Automatic focus handling in expanded view
- **High Contrast**: Ensures adequate color contrast for readability
- **Alternative Text**: Comprehensive alt text for all images
- **Touch Friendly**: Optimized touch targets for mobile devices

## Integration Examples

### With State Management

Integrate cards with global state management:

```typescript
// Using Redux or Zustand
import { useDispatch, useSelector } from 'react-redux';
import { InteractiveCards } from '@/data/components/Cards/InteractiveCards';

const CardsWithRedux = () => {
  const dispatch = useDispatch();
  const cards = useSelector(state => state.cards.items);
  const loading = useSelector(state => state.cards.loading);

  const handleCardAction = (cardId: number, action: string) => {
    dispatch({ type: 'CARD_ACTION', payload: { cardId, action } });
  };

  if (loading) return <CardsSkeleton />;

  return (
    <InteractiveCards
      cards={cards}
      onCardClick={(card) => handleCardAction(card.id, 'view')}
    />
  );
};
```

### With API Integration

Connect cards to backend APIs:

```typescript
import { useQuery } from '@tanstack/react-query';
import { InteractiveCards } from '@/data/components/Cards/InteractiveCards';

const ApiIntegratedCards = () => {
  const { data: cards, isLoading, error } = useQuery({
    queryKey: ['cards'],
    queryFn: async () => {
      const response = await fetch('/api/cards');
      return response.json();
    }
  });

  if (isLoading) return <div>Loading cards...</div>;
  if (error) return <div>Error loading cards</div>;

  return <InteractiveCards cards={cards} />;
};
```

const LazyCards = ({ cards }) => {

## Best Practices

1. **Image Consistency**: Use consistent aspect ratios and image quality
2. **Content Length**: Keep titles and descriptions concise but informative
3. **Loading States**: Implement skeleton loading for better perceived performance
4. **Error Handling**: Provide fallbacks for missing images or failed data loads
5. **Responsive Design**: Test across different screen sizes and orientations
6. **Performance**: Implement lazy loading for large datasets
7. **Accessibility**: Ensure all interactive elements are keyboard accessible
